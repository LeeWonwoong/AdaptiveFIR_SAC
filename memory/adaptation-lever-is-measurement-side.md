Two `tools/adapt_signal.py` diagnostics (2026-07-07, on `data_diag`) established where the (N,λ) adaptation lever exists in this UWB-4-range FME (self_anchor=False, aux-EKF-anchored, innov_gate=2.0).

**C (persistent single-anchor NLOS bias, DROPPED):** below-gate constant bias is a steady-state corruption; λ/N are TEMPORAL knobs and can't per-anchor reweight, so headroom ≈ 2%, corr ≈ 0.1. Only excludable (gate-tripping) faults are adaptable — and those just recreate anchor_dropout.

**3-tier IIR<FIR<AFIR study (recurrent_transient / varying_noise / recurrent_dropout):**
- **recurrent_transient (plant accel bursts)**: bursts kick |v| 0.01→1.2 m/s but localization error barely moves (peak 0.18 m, recovery ~0 s) because 4 UWB ranges keep position directly observed. optimal N does NOT shift (ΔN≈+0.1). Plant-side disturbances are a WEAK lever for UWB localization.
- **varying_noise**: optimal N is bimodal but SATURATES at N_max=20 (noise averaging) almost everywhere; nominal↔high-σ N distributions nearly identical (ΔN≈+0.3). Lever exists but N-range-capped → small swing.
- **recurrent_dropout**: clearest MECHANISM (FME excludes the missing anchor row; large N holds pre-dropout 4-anchor data) but the plain EKF baseline NaN-DIVERGES on missing anchors (~50% of dropout steps) — its RMSE is computed only over surviving finite steps, so the IIR comparison there is INVALID until the EKF skips missing channels.

**Cross-scenario numbers:** corr(‖ν‖, optimal N) ≈ +0.22–0.24 (just under the 0.3 bar) in ALL three. AFIR reliably beats FIR by ~20% RMSE / ~25% ITAE (per-step λ/N micro-optimization), BUT FIR LOSES to IIR/EKF on the clean scenarios (FME is baseline-noisier: EKF ~0.05 vs best-fixed-FME ~0.09 m) — so the "IIR<FIR" first link FAILS as tuned.

**Implication / next:** to make the IIR<FIR<AFIR chain hold, (1) fix the EKF baseline to handle missing anchors (fair IIR), (2) make disturbances measurement-side and HARSH enough to shift N_opt (gate-tripping NLOS outliers, deeper/longer dropouts, or a fixed-Q EKF that genuinely lags). recurrent_dropout is the most promising once the baseline is fixed; recurrent_transient (plant-side) is structurally weak for UWB localization.

ROOT CAUSE: **FIR beats IIR only against PROCESS-model error the recursive filter ACCUMULATES; measurement-side faults (dropout/NLoS) are exactly what a Kalman filter is built for**, so they don't yield IIR<FIR — except NLoS partially, only because the EKF's fixed R over-trusts it. So "lever is measurement-side" (for N-adaptation) and "FIR<IIR needs process-side error" are in TENSION. Only NLoS shows AFIR>IIR. Options to pursue: harsher NLoS (bias 0.8–1.5, still <gate 2.0; or 2 anchors) to make EKF overtrust catastrophic; or reintroduce persistent PROCESS-side mismatch (wrong mass/wind) as the real FIR<IIR driver; or reframe the paper around NLoS + "AFIR = best-of-both".
